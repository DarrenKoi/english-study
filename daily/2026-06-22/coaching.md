## 영어 다듬기

### 카드 1 — Codex 의견 묻기
- 내가 쓴 영어: "what codex thinks about this competition method? is it well established?"   (출처: transcript:[user] auto_recipe_creator)
- 정정: 의문문인데 평서문 어순입니다. `what codex thinks` → **`what does Codex think`** — 직접 의문문은 조동사 `do`를 끌어와 `What does + 주어 + 동사원형` 어순이 돼야 합니다. (`what Codex thinks`는 "코덱스가 생각하는 것"이라는 명사절이라 단독 질문으로 못 씁니다.) 고유명사 `Codex`는 대문자.
- 더 나은 표현: "What does Codex make of this competition method — is it a well-established technique?"  (`make of ~` = "~을 어떻게 보다/평가하다"로, 의견을 물을 때 `think about`보다 한결 자연스럽고 격식 있는 구어 표현)
- 왜: 질문 어순(do-지원)을 바로잡고, `think about` → `make of`로 한 단계 매끄럽게 올렸습니다. 뒤 절은 형용사 서술이라 `well established`로도 맞지만, `a well-established technique`처럼 명사를 꾸밀 땐 하이픈을 넣습니다.

### 카드 2 — 파일 분리 규칙이 이해가 안 됨
- 내가 쓴 영어: "can we be more specific about file trees? I do not understand the rule of separting files like monitor, vision, vlm"   (출처: transcript:[user] auto_recipe_creator)
- 정정: `separting` → **`separating`** (철자 오타 — separate 의 -ing 형). 그 외 문법 오류는 없습니다.
- 더 나은 표현: "Could we be more specific about the file tree? I don't follow the rule for splitting files into folders like monitor, vision, and vlm."
- 왜: ① 부탁은 `Can` 보다 `Could`가 약간 더 정중합니다. ② "이해가 안 된다"는 `I don't understand`도 맞지만 `I don't follow`가 "논리를 못 따라가겠다"는 뉘앙스로 더 자연스럽습니다. ③ `the rule of separating` → `the rule for splitting ... into ...`: 규칙의 *목적*을 말할 땐 `rule for -ing`가 관용적이고, 파일을 폴더로 "나눈다"는 맥락에선 `split into`가 핵심입니다. ④ 나열 끝에 `and`를 넣어 마무리.
