매일 밤 ~/Codes 전 프로젝트와 Claude Code 대화 로그에서 영어를 자동 수집해, 그날의 학습 다이제스트(daily/)와 누적 표현 노트(notes/)로 가공하고 git push 한다. 평소 안 쓰고 남는 토큰을 학습에 돌린다.

3. 데이터 파이프라인 (3 단계)

단계 0 — 진입 (run.sh → run.py)

run.sh 는 launchd 의 최소 env 에서 호출되므로 PATH·PYTHONPATH 를 명시한 뒤 uv run python -m pipeline.run 을 실행한다(uv 가 Python·venv·의존성 자동 관리). 먼저 자기 repo 를 git pull --ff-only.

info = collect(root)
  └─ 새 항목 0개 또는 mode == "empty"  →  종료 (상태 전진 없음)
prompt = _build_prompt(root, info)        # mode 에 따라 process.md / review.md
out = _invoke_claude(prompt, root)        # claude -p --allowedTools Read,Edit,Write,Glob
  └─ 출력 비었으면(LLM 실패)  →  종료 (상태 전진 없음 → 다음 실행 재시도)
tokenlog.append_token_log(...)            # 토큰/비용 기록
finalize(root, today)                     # 상태 전진 + push
안전 속성: collect 가 비거나 LLM 호출이 실패하면 finalize 를 호출하지 않는다. 상태 원장은 finalize 안에서만 전진하므로, 실패한 입력은 다음 실행에서 그대로 재시도된다.

단계 1 — collect() : 수집 · 예산 · 중복 필터

4 종류 입력을 단위(unit) 리스트로 만든다. 수집 우선순위:

  1. spool 노트 (최우선) — spool/*.{md,txt}. 사용자가 직접 적은 질문
  2. 문서 폴더 — 각 프로젝트를 git pull 후, 깊이 무관하게 doc/docs/shared_docs 안의 .md/.txt 중 최근 recent_days(7일) 수정분만 최신순. node_modules·hidden·빌드 산출물 가지치기
  3. 트랜스크립트 — ~/.claude/projects/**/*.jsonl 의 저장된 오프셋 이후 새 메시지 (파일당 단위 1개)

단위는 batch.build_batch() 로 char_budget(200k자) 안에서 조립되며, 예산을 넘으면 건너뛰되 최소 한 단위는 무조건 포함한다.

증분(중복 제거) — 두 개의 원장

입력원장 (state)동일성 키키가 바뀌는 조건
문서progress.json → docs_seenproject/relpath:mtime:size파일 수정(mtime/크기 변동) → 재수집
트랜스크립트progress.json → transcripts줄 오프셋새 줄 추가 → 그 줄부터 수집

_doc_key() 는 stat 만으로(mtime+size) 동일성을 판정해 변경 없는 파일은 읽지도 않는다. 원장은 finalize 성공 시에만 전진하므로 LLM 실패 후 재실행은 같은 문서를 다시 집는다.

빈 날 폴백 (2 단계)

정상 패스(mode="normal")에서 소비 0개면:

① backlog (mode="backlog")
   recent_days → backlog_days(14일)로 넓혀 7일 너머 미처리 문서 따라잡기
        │ 그래도 0개면
        ▼
② review (mode="review")
   notes/index.md 의 미복습 표현(reviewed.json 추적)을 오래된 순으로 복습 단위 1개로 묶음
        │ 그래도 0개면
        ▼
   mode="empty"  →  run.py 즉시 종료

매니페스트 출력

  • state/batch-<날짜>.md — LLM 입력 (provenance 헤더로 구분된 원문 묶음)
  • state/consumed-<날짜>.json — 소비된 단위만 기록. transcripts(파일→오프셋), spool, docs(경로→키), reviewed, mode, deferred. collect→finalize 사이 유일한 핸드오프

단계 2 — claude -p : LLM 가공

process.md 지침 요약 — ① 표현 추출 ② 레지스터 분류(professional/conversational/casual/technical) ③ 맥락 설명 ④ 정독 단락(문법 in context) ⑤ 격식 짝(refined↔plain) ⑥ 유사어·반의어 ⑦ 한글→영어 코칭(내가 쓴 한국어 우선, 어시스턴트 고급 문장 보조 2~4문장).

산출물: daily/<날짜>/{new-expressions,reading,coaching,digest}.md + spool 답변 spool/<이름>.md + notes/by-register/*.md(누적) + notes/index.md(복습 큐). 복습 모드(review.md)는 새 표현을 만들지 않고 notes/by-register 의 맥락으로 review.md+digest.md 복습본을 만든다.

단계 3 — finalize() : 상태 전진 · 정리 · push

consumed-<날짜>.json 을 읽어 소비한 만큼만 전진:

  1. 원장 전진 — progress.json 의 transcripts(오프셋)·docs_seen(키)·last_run 갱신
  2. 복습 전진 — 소비된 reviewed 표현을 reviewed.json(표현→날짜)에 누적
  3. spool 아카이브 — 처리 노트를 spool/done/<날짜>-<이름> 으로 이동(멱등성: 충돌 시 -2,-3…)
  4. 오래된 daily prune — today 기준 20일 넘은 daily/<날짜>/ 삭제(경계 보존)
  5. 죽은 index 링크 정리 — 링크 대상이 사라졌고 이미 복습한 표현 줄만 삭제. 미복습은 죽은 링크여도 보존(복습 큐 유지; 맥락 정본은 by-register 에 있음)
  6. commit & push — study: nightly digest <날짜>

7. 직접 쓰는 법

  1. 궁금한 것을 spool/ 에 .txt/.md 로 적어두면 다음 실행이 최우선으로 답한다 (답: daily/<날짜>/spool/<이름>.md)
  2. 아침에 daily/<오늘날짜>/digest.md 를 읽는다
  3. 토큰/비용 추이는 state/token-log.jsonl
  4. 누적 표현 notes/by-register/*.md, 복습 큐 notes/index.md
